Documentation

rich-html-editor is a framework-agnostic HTML editor built for one specific goal:

Allow non-technical users to edit existing HTML templates safely ‚Äî without breaking layout, CSS, or structure.

What problem does it solve?

Most WYSIWYG editors treat content as free-form text. This works for blogs ‚Äî but fails for:

  • Email templates
  • Marketing HTML
  • CMS-managed layouts
  • Approval-based workflows
  • Enterprise internal tools

Users can accidentally delete tags, change layout, or inject invalid HTML.

rich-html-editor takes a different approach.

How rich-html-editor is different

  • Template-first ‚Äî works on existing HTML, not blank pages
  • Iframe-based isolation ‚Äî no CSS or DOM leakage
  • Controlled editing ‚Äî only intended regions are editable
  • Safe HTML output ‚Äî editor artifacts removed before export
  • Framework-agnostic ‚Äî React, Angular, Vue, or Vanilla

What this is NOT

rich-html-editor intentionally avoids being a page builder or a free-form WYSIWYG editor.
  • ‚ùå No drag-and-drop layout editing
  • ‚ùå No arbitrary DOM manipulation
  • ‚ùå No inline CSS chaos
  • ‚ùå No uncontrolled HTML editing

These constraints are what make the editor safe and predictable.
